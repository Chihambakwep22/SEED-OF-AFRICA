# Render Deployment Configuration

## Backend (Django API)

**File: `render.yaml`** (in project root)
```yaml
services:
  - type: web
    name: seed-of-africa-api
    env: python
    buildCommand: |
      pip install -r requirements.txt
      python manage.py collectstatic --noinput
      python manage.py migrate
    startCommand: |
      gunicorn seedofafrica.wsgi:application --bind 0.0.0.0:$PORT
    envVars:
      - key: DEBUG
        value: "false"
      - key: SECRET_KEY
        generateValue: true
      - key: ALLOWED_HOSTS
        value: "seed-of-africa-api.onrender.com"
      - key: DATABASE_URL
        fromDatabase:
          name: seedofafrica_db
          property: connectionString
      - key: CORS_ALLOWED_ORIGINS
        value: "https://seed-of-africa-frontend.onrender.com"
      - key: DEFAULT_FROM_EMAIL
        value: "noreply@seedofafrica.com"
      - key: EMAIL_HOST
        value: "smtp.sendgrid.net"
      - key: EMAIL_PORT
        value: "587"
      - key: EMAIL_USE_TLS
        value: "true"
      - key: EMAIL_HOST_USER
        value: "apikey"
      - key: EMAIL_HOST_PASSWORD
        generateValue: true
    healthCheckPath: /api/health/
    autoDeploy: true

databases:
  - name: seedofafrica_db
    type: postgres
    env: postgres
    plan: free
    databaseName: seedofafrica
    user: seedofafrica_user
```

**Environment Variables for Render Backend:**
- `DEBUG=false`
- `SECRET_KEY=<auto-generated>`
- `ALLOWED_HOSTS=seed-of-africa-api.onrender.com`
- `DATABASE_URL=<from Render database>`
- `CORS_ALLOWED_ORIGINS=https://seed-of-africa-frontend.onrender.com`
- `DEFAULT_FROM_EMAIL=noreply@seedofafrica.com`
- `EMAIL_HOST=smtp.sendgrid.net`
- `EMAIL_PORT=587`
- `EMAIL_USE_TLS=true`
- `EMAIL_HOST_USER=apikey`
- `EMAIL_HOST_PASSWORD=<your-sendgrid-api-key>`

## Frontend (React)

**File: `frontend/render.yaml`** or use Render's static site config:
```yaml
buildCommand: npm install && npm run build
outputDirectory: dist
envVars:
  - key: VITE_API_URL
    value: https://seed-of-africa-api.onrender.com/api
  - key: NODE_ENV
    value: production
```

**Or create `frontend/Dockerfile`:**
```dockerfile
FROM node:18-alpine as build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

**File: `frontend/nginx.conf`:**
```nginx
server {
    listen 80;
    server_name _;
    
    root /usr/share/nginx/html;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass https://seed-of-africa-api.onrender.com/api;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
    
    # Static file caching
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

## Database Migration Steps:
1. Create PostgreSQL database on Render
2. Update `settings.py` to use PostgreSQL (already configured above)
3. Run migrations: `python manage.py migrate`
4. Create superuser: `python manage.py createsuperuser`
5. Load initial data for `ImpactMetric` model
